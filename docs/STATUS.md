# Rabbitbone status

Rabbitbone is currently at `0.0.3.3`.

The system now boots as a UEFI x86_64 live ISO. The default image contains a freestanding `EFI/BOOT/BOOTX64.EFI` loader, the Rabbitbone kernel, and a live root disk image. The loader reads the kernel and root image from the ISO EFI system partition, converts the UEFI memory map into bootinfo v2, passes the root image as a boot module, passes the UEFI GOP framebuffer metadata, exits boot services, and jumps into the independent kernel. The kernel registers that module as a writable RAM-backed block device, maps the framebuffer for VMware's graphical console, probes its MBR Linux partition, mounts the seeded EXT4 filesystem at `/disk0`, and starts `/disk0/sbin/init` with `/disk0/bin/rbsh` as the shell path.

## 0.0.3.3 status

UEFI live boot is the release path. `make image` and `make` now produce `build/rabbitbone-live.iso`; the raw BIOS disk image has moved to `make legacy-image` for regression comparison. The ISO builder is self-contained and creates an El Torito UEFI boot image containing `/EFI/BOOT/BOOTX64.EFI`, `/RABBITBONE/KERNEL.BIN`, and `/RABBITBONE/ROOT.IMG`. The root image is generated by `rabbitbone-install --root-only`, preserving the existing EXT4 seed layout and all disk-backed userland files without embedding the userland blob into the kernel.

Bootinfo remains version 2 and now reports `source=uefi-live-iso`, UEFI/live-root flags, kernel size in sectors, boot-module count/address, and the kernel command line through the `boot` diagnostic. The boot RAM disk driver exposes the loaded root image as `ramdisk0`, so the existing MBR/EXT4 mount selection path remains intact.

The `0.0.3.3` patch hardens source-level safety checks around path normalization, syscall validation, auth secrets, pipe/file lifetime wrappers, shell redirections, and Rust ABI synchronization while keeping the UEFI live ISO release path unchanged.

## 0.0.2.14 status

Disk-backed userland is implemented. The release kernel no longer links the full `/bin` and `/sbin` payload blob by default; the installer places user ELF programs, `/sbin/init`, `/bin/sh`, and `/etc/profile` directly into the EXT4 partition. The kernel mounts the selected EXT4 partition at `/disk0`, starts `/disk0/sbin/init`, and keeps small `/bin`/`/sbin` compatibility symlinks for existing absolute-path tests. The default PATH is `/disk0/bin:/disk0/sbin:/bin:/sbin`.

## 0.0.2.12 status

- Added ACPI RSDP/RSDT/XSDT discovery with MADT and HPET table parsing.
- Added LAPIC/IOAPIC topology diagnostics while keeping PIC/PIT as the interrupt fallback.
- Added HPET free-running counter initialization, invariant-TSC calibration when HPET is available, and timer diagnostics.
- Added SMP bootstrap groundwork: bounded CPU topology, BSP identification, and AP-startup staging metadata without enabling multi-core scheduling yet.
- Added shell/kctl diagnostics: `acpi`, `apic`, `hpet`, `timer`, and `smp`.

## Working now

- UEFI live ISO boot path with a freestanding `BOOTX64.EFI` loader, bootinfo v2 handoff, and GOP-backed framebuffer console.
- Boot-module RAM disk registration and EXT4 mount from the live `ROOT.IMG` at `/disk0`.
- Retained Legacy BIOS stage1/stage2 path as a non-default `legacy-image` regression target.
- x86_64 GDT, IDT, IRQ, paging, VMM with kernel-image W^X/NX page protections, heap, panic and logging code.
- Writable ramfs at `/`, devfs at `/dev`, tarfs support, and partition-bounded writable EXT4 mounted from the first Linux MBR partition.
- PCI config-space enumeration, AHCI SATA probing/read-write support, ATA PIO fallback, MBR parsing, and a driver-tagged block registry used by EXT4 mount selection.
- ELF64 user program loading, kernel `#!` interpreter dispatch for scripts, ring3 entry, `int 0x80` syscalls, `/disk0/sbin/init`, and `/disk0/bin/rbsh`.
- Process records, file descriptors, async `spawn`, blocking `wait` and `sleep`, copy-on-write `fork`, per-process `brk`/`sbrk` heaps, VMA-backed demand-paged anonymous, file-backed private, and anonymous `MAP_SHARED` `mmap`/`munmap`/`mprotect`, and `exec`/`execv`/`execve`, including shebang scripts with bounded recursion.
- Single-core preemptive scheduling through PIT IRQ0, with ACPI-discovered APIC topology and HPET/TSC clocksource diagnostics staged for later APIC timer routing.
- Rust safety-boundary modules linked into the kernel for syscall dispatch, usercopy, VFS routing, and path policy checks, with generated ABI constants shared from the C header.
- Host tests and in-kernel `ktest`.

## Not done yet

- Persistent installation from the live ISO; the current ISO root is RAM-backed and intentionally loses writes after reboot.
- Full Linux-compatible EXT4 journal descriptor/commit block format, extent tree depth above 2, htree hash variants beyond the Rabbitbone-compatible index block, full delayed allocation/writeback-daemon policy, app/package rollback database, deeper fsck repair coverage, and metadata checksum coverage matching every upstream EXT4 feature flag.
- AP startup/multi-core scheduling, IOAPIC IRQ routing, APIC timer, MSI/MSI-X, PCIe ECAM, NVMe, USB, and GUI.
- Real entropy and a future cryptographic `/dev/random`; current `/dev/prng` and `/dev/urandom_insecure` are explicitly non-cryptographic.

## Runtime checks

After building `build/rabbitbone-live.iso`, boot with `vmware/rabbitbone-uefi-live.vmx` or attach the ISO as a UEFI CD/DVD device. Then run:

```text
boot
ktest
```

A passing run ends with:

```text
KTEST_STATUS: PASS
```

Useful manual checks:

```text
mounts
ls /
cat /etc/motd
ls /disk0
cat /disk0/hello.txt
run /bin/hello
run /bin/fscheck /disk0/hello.txt
run /bin/execcheck
sched
preempt
pci
disks
blk
```

- `mmap` VMAs are demand-paged: anonymous and file-backed private pages are materialized on user page fault, while `munmap`, `mprotect`, fork/COW, exec, and exit keep VMA/PTE/refcount state consistent.

- UEFI NXE handoff is now covered by kernel CPU bring-up, so the UEFI path can safely load NX-marked kernel page tables.
- UEFI framebuffer output is active after paging init and renders printable ASCII directly; serial output remains enabled for full boot logs.
