# Security Policy

## Supported Versions

Rabbitbone is experimental OS development software. Security fixes are handled on the current `main` branch only.

## Reporting a Vulnerability

Please do not publish a working exploit before maintainers have had a chance to respond. Report suspected vulnerabilities through GitHub's private vulnerability reporting flow if it is enabled for the repository. If that is not available, contact a project maintainer privately through the maintainer contact channels listed on the GitHub organization or repository profile.

Useful reports include:

- The affected component or subsystem.
- Steps to reproduce the issue.
- The expected and actual behavior.
- Any relevant logs, crash output, or test case.
- Whether the issue affects the host build tools, the kernel, the filesystem stack, or userland.

Maintainers will acknowledge valid reports as soon as possible and coordinate a fix on the current development branch.
